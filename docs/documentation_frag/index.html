FragPELE Parameters¶

Compulsory flags¶

Frag PELE grows an atom onto a core in N growing steps while moving protein and ligand. Afterwards a final sampling simulation is run to fully explore the ligand-protein conformational space.

  • frag_core: Core of the molecule we want to add fragments to. Required parameter

Optative flags¶

  • growing_steps: Number of steps to grow the fragment with.

  • steps_in_gs: Number of pele steps within each growing step

  • sampling_steps: Number of pele steps in the final sampling simulation

  • protocol: Type of protocol. options = [HT, ES]. For more info please refere here.

  • cpus: Cpus to use

growing_steps: 6
steps_in_gs: 6
sampling_steps: 20
protocol: HT
cpus: 24

Methods¶

Low-thrughput¶

Ideal when performing lead optimization. Only a few but very well thought changes into the core. Here you must manually specify what and where to grow into the serie_file. For more information about the input file please refere here.

  • fag_input: Frag pele input. For more please check here

frag_core: "/home/daniel/PR_core.pdb"
fag_input: "/home/daniel/serie_file.conf"

High-throughput¶

Ideal when running many fragments onto one core. Need a complex.pdb with protein and core and a sdf with a list of fully grown ligands. The input file will be generated automatically.

  • frag_ligands: full_ligands.sdf

frag_core: "/home/daniel/complex_and_core.pdb"
frag_ligands: "/home/daniel/ligands.sdf"
protocol: "HT"

High-throughput AI¶

Ideal when exploring new option in a highthroughput manner. The RNN will first create the ligands. Then fragPele will dinamically grow and rank them. Reinforcement learning may be implemented in future versions.

frag_core: "/home/daniel/complex_and_core.pdb"
frag_ai: true
protocol: "HT"
